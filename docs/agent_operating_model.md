# FreightTrigger Agent Operating Model

FreightTrigger is operated as a faceless AI logistics sales intelligence engine.
The user owns the business rails. The engine handles research, scoring, reporting,
client targeting, outreach strategy, and follow-up logic.

## The 8-Agent Team

1. **CEO/Mastermind Agent**
   Owns positioning, vertical selection, pricing, exclusion rules, and scale/kill decisions.

2. **Signal Acquisition Agent**
   Searches public/business sources for freight-relevant movement and evidence URLs.

3. **Shipper Scoring Agent**
   Scores urgency, freight relevance, confidence, and timing without claiming verified intent.

4. **Contact/Org Agent**
   Identifies likely buyer roles and contact paths without overclaiming named decision-makers.

5. **Lane/Need Intelligence Agent**
   Infers likely freight/service needs such as reefer, FTL, LTL, warehousing, drayage, final mile, or forwarding.

6. **Outbound Angle Agent**
   Produces source-specific email/call/LinkedIn hooks and follow-up positioning.

7. **Strategic Edge Agent**
   Detects underserved verticals, territory patterns, and repeatable signal types.

8. **Compliance/Audit Agent**
   Enforces source-backed evidence, opt-out/suppression handling, no guaranteed intent claims, and no freight operations language.

## Current Engine Sequence

1. Search public signal candidates.
2. Extract source content.
3. Classify freight relevance.
4. Score urgency/confidence.
5. Infer freight need and buyer path.
6. Write Airtable records.
7. Generate report-ready summaries.
8. Prepare outreach angles.

## 24/7 Target State

- Hourly source scans.
- Daily hot-signal shortlist.
- Weekly paid report generation.
- Gmail API reply classification.
- Suppression list enforcement.
- Stripe subscription status checks.
- Conversion intelligence tracking.

## Non-Negotiable Boundaries

- FreightTrigger is sales intelligence only.
- No brokerage, load handling, shipment management, carrier selection, invoices, contracts, payments, or private freight documents.
- Public site stays high-level; internal docs/scripts hold the operating playbook.
